# ✅ 工作正常的聊天页面

## 🎯 推荐使用页面

**simple-chat.html** - 简化但功能完整的聊天页面

```
访问地址: http://localhost:8080/simple-chat.html
```

## ✨ 特点

1. **简化架构** - 去除了可能导致问题的复杂功能
2. **清晰日志** - 控制台输出详细的连接过程
3. **自动重连** - 连接断开后自动重连
4. **美观界面** - 现代化的渐变设计
5. **状态指示** - 实时显示连接状态

## 🔧 技术细节

### 连接流程

1. **建立 WebSocket 连接**
   ```
   ws://localhost:18789
   ```

2. **等待 challenge**
   ```
   事件: connect.challenge
   包含: nonce
   ```

3. **发送认证消息**
   ```json
   {
     "type": "req",
     "method": "connect",
     "params": {
       "auth": { "token": "..." }
     }
   }
   ```

4. **认证成功**
   ```
   事件: connect.ok
   状态: 已连接
   ```

### 消息发送

```javascript
{
  "type": "req",
  "method": "chat.send",
  "params": {
    "sessionKey": "agent:main:main",
    "message": "用户消息",
    "idempotencyKey": "唯一ID"
  }
}
```

### 消息接收

```javascript
{
  "type": "event",
  "event": "chat",
  "payload": {
    "state": "final",
    "message": "回复内容",
    "runId": "...",
    "seq": 1
  }
}
```

## ⚠️ 故障排查

### 问题: 连接失败

**检查控制台错误:**
```
按 F12 → Console 标签 → 查找红色错误
```

**常见错误:**
1. `ERR_CONNECTION_REFUSED` → Gateway 未运行
2. `认证失败` → Token 错误
3. `超时` → 网络问题

### 问题: 消息发送失败

**确认:**
1. 状态是否为 "已连接"（绿色）
2. 控制台是否有错误
3. Gateway 是否正常运行

### 问题: 自动重连失败

**手动重连:**
```javascript
// 在控制台中运行
location.reload();
```
